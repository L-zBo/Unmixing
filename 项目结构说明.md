# 项目结构说明

这份文档是给“第一次看这个仓库的人”准备的，目标只有一个：

- 说清楚每个文件夹是干什么的
- 说清楚哪些是主线，哪些是历史归档
- 避免把现在还在用的主线代码和旧路线代码混为一谈

## 1.根目录各项分别是什么

### `README.md`

项目总入口。

作用：

- 快速说明项目现在主线是什么
- 给出目录总览
- 告诉你该先看哪份文档

如果你只想先搞明白项目做什么，先看它。

### `AGENTS.md`

这是协作规则文件，不是业务代码。

作用：

- 规定提交和推送习惯
- 规定中文回复、文件改动边界等协作要求
- 严禁空 commit 与 prompt 原文 message

### `项目结构说明.md`

就是你现在看的这份文件。

作用：

- 用更直白的方式解释仓库结构
- 说明每个目录的职责

### `.gitignore`

告诉 git 哪些内容不要提交。

目前最重要的两条是：

- `dataset/` 不进 git
- `outputs/` 不进 git

## 2.顶层目录怎么分

仓库已经收成「按职责扁平铺开」的结构，主线代码不再藏在 `src/demixing/` 下面，而是分散到各职责顶层目录。

### `dataset/`

原始数据目录。

作用：

- 存 PE、PP、淀粉纯谱
- 存真实混合拉曼面扫数据
- 存测试集和泛化集

原则：

- 只读
- 不覆盖
- 不移动
- 不提交

### `outputs/`

实验产物目录。

作用：

- 放预处理结果
- 放合成真值数据
- 放单图/批量实验结果
- 放各种图、表、summary

原则：

- 只存产物
- 不进 git

### `docs/`

详细路线文档目录。

按路线拆开：

- `[docs/nnls_unmixing_flow.md](f:/VsCodeproject/Unmixing/docs/nnls_unmixing_flow.md)`
  当前主线文档，讲 NNLS 解混路线。
- `[docs/legacy_classification_flow.md](f:/VsCodeproject/Unmixing/docs/legacy_classification_flow.md)`
  旧路线归档，记录早期“家族分类+组级空间评估”那套方案。

### `preprocessing/`

端元加载与协议化预处理模块。

主要文件：

- `[preprocessing/preprocess.py](f:/VsCodeproject/Unmixing/preprocessing/preprocess.py)`
  支持 `als_l2`、`als_max`、`none_l2` 三种预处理协议
- `[preprocessing/endmembers.py](f:/VsCodeproject/Unmixing/preprocessing/endmembers.py)`
  纯谱加载、按协议构建端元矩阵
- `[preprocessing/preprocess_dataset.py](f:/VsCodeproject/Unmixing/preprocessing/preprocess_dataset.py)`
  命令行入口，把 `dataset/` 处理到 `outputs/preprocessing/`

### `synthetic/`

合成真值数据相关。

主要文件：

- `[synthetic/generator.py](f:/VsCodeproject/Unmixing/synthetic/generator.py)`
  从端元谱出发生成有像素级真值的二维丰度图与混合谱
- `[synthetic/generate_dataset.py](f:/VsCodeproject/Unmixing/synthetic/generate_dataset.py)`
  命令行入口，生成 `outputs/synthetic_unmixing/`

### `unmixing/`

经典解混核心模块。

主要文件：

- `[unmixing/unmix.py](f:/VsCodeproject/Unmixing/unmixing/unmix.py)`
  含 `OLS`、`NNLS`、`FCLS`、盲 `NMF` 与端元对齐逻辑

### `visualization/`

主线解混路线的可视化模块，按图类型分子目录。

子目录：

- `abundance/` 丰度热图
- `residual/` 残差空间图
- `reconstruction/` 输入谱与重构谱对比
- `method_comparison/` 方法横向对比柱图
- `preprocessing/` 预处理协议对比三视图与丰度网格

顶层 `[visualization/__init__.py](f:/VsCodeproject/Unmixing/visualization/__init__.py)` 一次性 re-export 全部 8 个绘图函数；公共工具放在 `_common.py`。

### `experiments/`

正式实验入口脚本，平铺。每个脚本对应一个明确的实验场景：

- `run_real_unmixing_single.py` 单张真实面扫图
- `run_real_method_comparison.py` 单图四方法对比
- `run_batch_method_comparison.py` 多图四方法批量
- `run_synthetic_method_comparison.py` 合成真值四方法对比
- `run_real_preprocessing_comparison.py` 单图三协议对比
- `run_batch_preprocessing_comparison.py` 多图三协议批量
- `run_generalization_batch.py` 跨淀粉源泛化批量

### `utils/`

通用工具模块。

主要文件：

- `[utils/io.py](f:/VsCodeproject/Unmixing/utils/io.py)`
  `save_predictions` 与 `save_experiment_summary`，统一实验产物落盘格式

### `archive/`

历史归档区，不参与当前主线。

子目录：

- `archive/legacy_classification/` 旧家族分类路线（v1~v5、external_test）
- `archive/legacy_data_pipeline/` 旧 manifest 与 quality 数据管线
- `archive/legacy_training/` 旧训练相关脚本

它们仍在使用旧的 `from demixing.X` import 路径，因为已经不进入主线 import 链；如果将来需要复活旧路线，需要单独迁移其 import。

### `.spec-workflow/`

这是工作流模板/辅助目录，不是项目主业务代码。

## 3.docs 里两份文档怎么理解

### 当前主线：`docs/nnls_unmixing_flow.md`

对应现在真正要做的路线：

- 默认预处理主线：`ALS + L2`
- 默认解混主线：`NNLS`
- 对比方法：`CLS/OLS`、`FCLS`、`NMF`
- 真实数据验证 + 合成真值验证

如果你现在的论文、代码、实验都围绕 NNLS 展开，主要看它。

### 历史路线：`docs/legacy_classification_flow.md`

对应旧方案：

- 家族分类
- 组级/空间级评估
- CNN/SVC 这些早期路线

它的价值不是继续当主线，而是：

- 记录项目怎么一步步走到现在
- 给当前解混路线提供历史参照
